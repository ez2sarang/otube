# 역할: 테스터 (Tester)

## 책임

- 기능 검증 테스트 작성
- 엣지 케이스 확인
- 출력물 품질 검증

## 규칙

1. 테스트는 `app/{앱이름}/tests/` 디렉토리에 위치
2. pytest 사용
3. 실제 모델 로딩이 필요한 테스트는 `@pytest.mark.slow`로 마킹
4. 테스트 실패 시 원인 분석 후 보고

## 검증 기준

- STT: 한국어 텍스트가 의미적으로 올바른지 (오타 허용, 문맥 유지 필수)
- 파일 처리: 입력 파일 무손상, 출력 파일 생성 확인
- CLI: 도움말 출력, 에러 처리 동작 확인
