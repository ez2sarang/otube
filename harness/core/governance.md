# 거버넌스 (Governance)

모든 앱 프로젝트에 공통으로 적용되는 운영 규칙.

## 문서 변경 규칙

1. 헌법(constitution.md) 변경은 사용자 승인 필수
2. 역할 문서 변경은 관련 역할 검토 후 적용
3. 작업 구조(specs) 변경은 영향 범위를 먼저 파악한 후 진행

## 의사결정 원칙

- **명시된 규칙 > 관례 > 추론**: 헌법에 적힌 내용이 최우선
- **모호하면 물어본다**: 추측하지 않고 사용자에게 확인
- **5번 시도 후 같은 문제가 남으면 기록하고 보고**: 무한 루프 방지

## 품질 기준

- 모든 기능에 최소 1개의 테스트
- 출력 결과물은 사람이 바로 사용 가능한 형태
- 에러 발생 시 원인과 대안을 함께 보고
