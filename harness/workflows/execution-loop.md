# 실행 루프 (Execution Loop)

모든 작업은 다음 사이클을 따른다.

## 기본 루프

```
1. 작업 지시 확인 → specs/ 문서 참조
2. 구현 → coder 역할 실행
3. 검증 → tester 역할 실행
4. 리뷰 → reviewer 역할 실행
5. 피드백 반영 → 사용자 확인
6. 통과 시 커밋 / 미통과 시 2번으로 복귀
```

## 반복 제한

- 같은 이슈로 5번 이상 반복하면 멈추고 사용자에게 보고
- 보고 형식: "문제 설명 / 시도한 방법 / 추천 대안"

## 체크포인트

- 의미 있는 진전이 있을 때마다 git commit
- 커밋 메시지에 현재 상태 간략 기술
